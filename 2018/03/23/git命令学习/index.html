分支创建和管理

  • 创建分支git checkout -b dev

  • 查看当前所有分支 git branch，星号表示当前分支

  • 切换分支 git checkout master

  • 分支内容合并到master上 git merge dev

  • 删除分支 git branch -d dev

总结：

  1. git是分布式版本控制系统
  2. git克隆项目以后可以在本地看的所有log
  3. git能在不联网的情况下合并分支提交代码
  4. git可以多个版本库，和svn不同，只有一个版本库存放所有资源
  5. git的上传拉去代码速度快，svn的慢
  6. git所有分支基于一个项目，svn可以是某个目录下
